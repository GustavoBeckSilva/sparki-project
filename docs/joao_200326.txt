# Navegação via coordenadas

- Implementei um serviço de handle_navigate_request no edge que receberá os pontos de localização no mapa e já converte isso para comandos válidos de movimentação
- Também já estou consumindo esse serviço no backend via rosbridge e é possível fazer requisições de localizações via API
- O formato do corpo das requisições será de {x: float32, y: float32}, indicando as coordenadas que deverão ser passadas


# Exemplo de comando que pode ser utilizado direto no serviço:

- ros2 service call /navigate_to_waypoint sparki_interfaces/srv/NavigateToWaypoint "{x: 2.0, y: 0.0}"

(OBS: Deve estar num terminald dentro do docker)
